# BELLORE 디자인 표준 (이 문서를 벗어나지 말 것)

사장님 확정 지시: "규격·정렬이 딱 맞는 이 느낌을 표준으로 저장하고 절대 벗어나지 마라."
**그리고(2026-06): "레퍼런스(바이버/구구스)처럼 컬러로, 똑같이 가라" — 흑백화 금지.**
참고 레퍼런스: `design-refs/` 폴더의 바이버·구구스 화면(사장님이 직접 업로드).
화면별로 그 이미지를 보고 색·레이아웃·구성요소를 **그대로** 맞춘다.

## 0) 절대 원칙
- **규격/정렬 일관성이 최우선.** 같은 종류의 요소(입력칸·버튼·카드·라벨)는 높이·패딩·
  라운드·폰트·간격이 픽셀 단위로 동일해야 한다. 들쭉날쭉/줄바꿈 어긋남 금지.
- **여백을 충분히, 가운데 정렬 느낌.** 박스를 화면 끝까지 꽉 채우지 말고 좌우 여백을 줘
  중앙 컬럼(최대폭)에 정돈. (사장님: "박스가 너무 길어 → 좌우 여백 줘서 가운데 정렬")
- 새 UI를 만들 때 기존 클래스/토큰을 **재사용**한다. 임의 인라인 스타일·일회용 색상 금지.

## 1) 색 (레퍼런스 충실 — 컬러 허용, 2026-06 사장님 확정 변경)
> ⚠️ **방향 전환:** 이전의 "모노크롬 전용" 규칙은 폐기. 사장님 지시
> "레퍼런스(바이버/구구스)처럼 컬러 허용 — 똑같이 가라." 레퍼런스에 컬러
> 아이콘·컬러 버튼·컬러 배너가 있으면 **그 색을 그대로 살린다.** 색을 빼서
> 흑백으로 만들지 말 것. (이게 그동안 화면이 레퍼런스와 달라 보인 핵심 원인이었음.)
- 배경: 흰색 `#fff` / 페이지 베이스 `#f4f1ea`(기존 유지).
- 텍스트: 본문 `#1a1a1a`, 보조 `#6b6b6b`, 비활성 `#9a9a9a`.
- 보더: `#e5e3df`(연회색). 포커스 보더: `#1a1a1a`.
- **카테고리 원형 아이콘 = 컬러**(바이버 레퍼런스처럼). 연회색 원(`#f2f3f5`) +
  기능별 컬러 아이콘. 기본 팔레트:
  포인트/쿠폰 주황 `#ff8a00` · 내시계팔기 초록 `#16a34a` · 내관심 틸 `#14b8c4` ·
  최근본상품 파랑 `#2d7ff9` · 장바구니 보라 `#7c5cfc`. (새 항목은 이 톤 안에서 추가)
- 주요(Primary) 액션 버튼: 검정 `#111` 배경 + 흰 글씨(기존 CTA 유지). 단,
  레퍼런스가 컬러 CTA면 그 화면은 레퍼런스 색을 따른다.
- 알림 배지 등 카운트: 빨강 점/배지 OK(바이버처럼). 위험(탈퇴/삭제) 글씨는 절제.
  - ⚠️ 관리자 회원관리의 과한 빨간 글씨는 여전히 지양(보기 편하게).
- **규격·정렬 일관성 원칙은 그대로 유지**(색만 허용된 것이지, 들쭉날쭉 허용 아님).

## 2) 글꼴 (Pretendard 전체 통일 — 확정)
- 전 화면 본문/숫자/영문: `Pretendard, -apple-system, ...` 산세리프.
- 숫자: `font-variant-numeric: tabular-nums;` 로 정렬 맞추되 **작게 줄지 않게** 본문과
  동일 크기. Cormorant/Playfair 등 세리프를 UI 본문·숫자에 쓰지 말 것.
- 예외: 상단 'BELLORE' 워드마크/로고만 기존 세리프 유지 가능.

## 3) 입력 필드 (레퍼런스의 핵심 = 라벨+박스+우측 아이콘)
- 구조: **라벨(위, 작은 회색)** → **입력 박스(아래, 한 줄)**. 라벨은 항상 박스 위.
- 입력 박스: 높이 **52px**, 좌우 패딩 16px, 라운드 12px, 보더 1px `#e5e3df`,
  글자 16px. 포커스 시 보더 `#1a1a1a`(그림자 없음/약하게).
- **인라인 액션은 박스 안 오른쪽 끝**: 수정 연필 ✎ / 비밀번호 표시 눈 👁 / 지우기 ✕ /
  '인증하기' 버튼은 박스 우측에 정렬. (사장님: 비밀번호 표시 = 박스 맨 오른쪽 눈 표시)
- 같은 줄에 보조 버튼(예: 이메일 + 인증하기)이 오면 높이를 입력칸과 **정확히 동일(52px)**.

## 4) 버튼
- 1차 CTA: 풀폭, 높이 54px, 라운드 12px, 검정 배경/흰 글씨, 글자 16px 700.
- 2차/뒤로: 흰 배경 + `#e5e3df` 보더 + 검정 글씨, 같은 높이.
- 비활성: 배경 `#eceae6`, 글씨 `#9a9a9a`.
- 한 줄에 [2차][1차]가 나란히 올 때 높이·라운드 동일, 간격 10px.

## 5) 카드 / 섹션
- 카드: 흰 배경, 라운드 14px, 보더 1px `#e5e3df`(그림자는 아주 약하게 또는 없음).
- 섹션 제목: 16~18px 700, 아래 12px 간격.
- 리스트 행(소셜연동/메뉴): 풀폭 카드, [아이콘][라벨 ····· 상태칩/›] 좌우 양끝 정렬.
- 항목 간 간격 10~12px로 일정하게.

## 6) 단계형(토스식) 폼 화면
- 전체화면 페이지. 상단 '‹ 뒤로' + 제목.
- 본문 맨 위 **큰 굵은 질문 제목**(24~26px 700, 두 줄 가능).
- 그 아래 라벨+입력 1~N개. 하단 고정 풀폭 검정 CTA.
- 좌우 패딩 충분(모바일 20px, 와이드는 중앙 컬럼 최대폭 제한).

## 7) 레이아웃 폭/정렬
- 마이페이지·폼 등 내부 콘텐츠는 **중앙 정렬 + 최대폭(≈480~560px 폼, 720px 리스트)**.
- 화면을 꽉 채우지 말고 좌우 여백. PC에서 과한 가로폭 금지.
- PC에서 모달 내부 자체 스크롤 만들지 말고 **윈도우(페이지) 스크롤** 사용.

## 8) 체크리스트(새 화면/수정 시 매번 확인)
- [ ] 같은 요소 높이/패딩/라운드/폰트가 전부 동일한가
- [ ] 라벨은 박스 위, 액션 아이콘은 박스 안 오른쪽인가
- [ ] 색이 모노크롬(검정/흰/회색)인가, 파랑/그린/불필요한 빨강 없는가
- [ ] 글꼴 Pretendard, 숫자가 본문과 같은 크기인가
- [ ] 좌우 여백 + 중앙 정렬, PC에서 안 늘어지는가
- [ ] CTA는 풀폭 검정인가, 정렬 어긋남 없는가
