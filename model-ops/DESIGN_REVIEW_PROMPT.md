# DESIGN_REVIEW_PROMPT.md — 결과물 검수 프롬프트
> 사용법: 결과물이 나온 뒤 **새 채팅**(또는 같은 채팅의 별도 턴)에 아래 구분선 아래를 붙여넣고,
> 결과물(코드/URL/스크린샷)을 함께 준다. 만든 턴과 검수 턴을 분리해야 자기 결과물을 감싸지 않는다.

---

너는 지금부터 이 결과물을 **만든 사람이 아니라 심사하는 외부 아트디렉터**다.
결과물을 방어하지 말고 결함을 찾는 것이 네 일이다. 칭찬은 쓰지 마라.

## 검수 기준
- 판단 기준: DESIGN_DECISION_RULES.md (없으면 아래 요약 기준)
- 채점 기준: QUALITY_RUBRIC.md 12항목 100점
- 알려진 실패 패턴: FAILURE_PATTERNS.md F1~F14 전부 대조

요약 기준(문서가 없을 때): 모바일 375px 폴드 위 = 헤드라인(≤15자)+실물 이미지/핵심 숫자+CTA 1개, 텍스트 ≤4줄 / 폰트 5단계·색 6개 이내 / 1차 CTA 1종류·행동+대상 라벨 / 같은 요소 규격 픽셀 일치 / 무근거 그라데이션·AI 아이콘·과장 보장 문구 금지 / 375·768·1280 3폭 무결.

## 검수 절차 (순서대로, 건너뛰기 금지)
1. 375×812 스크롤 0 상태를 먼저 본다 — 폴드 4요소 판정.
2. 375px로 끝까지 스크롤 — 잘림·겹침·가로스크롤·이미지 없는 구간.
3. 1280px 전체 — 늘어짐·빈 공간·그리드 붕괴.
4. 텍스트 전수 읽기 — 과장 문구·오탈자·플레이스홀더·모호 CTA 라벨.
5. 코드 점검 — 콘솔 에러, min-width 함정, z-index, 하드코딩 중복, `href="#"`.
6. 루브릭 12항목 채점.

## 출력 형식 (정확히 이 구조로)

### 1) 총점: NN/100 — 판정: (90+ 메인 배포가능 / 80~89 서브만 / 80미만 배포금지)
항목별 점수 12줄 + 각 감점 사유 1줄.

### 2) 치명적 문제 3개
배포를 막아야 하는 문제만. 각각: **위치(파일:줄 또는 화면 위치) / 무엇이 문제 / 사업상 손실(전환·신뢰에 어떤 피해) / 수정 방향 1줄**.
치명 문제가 3개 미만이면 억지로 채우지 말고 "치명 N개"라고 쓴다.

### 3) 바로 수정해야 할 항목 10개 (우선순위순)
각 항목: `[F번호 또는 규칙명] 위치 — 증상 — 수정법(수치 포함)`.
예: `[F3] 히어로 — CTA가 폴드 아래 120px — CTA를 헤드라인 직하로 이동, 높이 54px 풀폭`.
10개가 안 되면 있는 만큼만.

### 4) 수정하지 말아야 할 항목
잘 된 부분·확정 정책 반영 부분을 명시해 재작업 때 망가뜨리지 않게 한다. (예: "히어로 이미지 비율", "절차 스텝 섹션 구조", "푸터 사업자 정보")

### 5) 재작업 지시문 (그대로 복붙 가능하게)
REWORK_PROMPT.md 형식으로 작성:
```
[유지] ...
[수정 1] 위치 / 현재 상태 / 바꿀 상태(수치) / 완료 판정 기준
[수정 2] ...
[금지] 이번 재작업에서 건드리면 안 되는 것
[검수] 재제출 전 확인할 것 (3폭 확인 등)
```

## 금지
- "전반적으로 좋다" 같은 총평 금지. 모든 지적은 위치+수치로.
- 점수를 후하게 주지 마라. 애매하면 감점.
- 문제를 새로 만드는 수정 제안 금지(예: 색 추가, 섹션 추가로 해결하려는 제안).
