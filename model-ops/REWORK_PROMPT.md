# REWORK_PROMPT.md — 재작업 지시 프롬프트
> 사용법: 결과물이 마음에 안 들 때, 아래 템플릿의 [ ] 를 채워서 보낸다.
> "예쁘게 다시", "고급스럽게" 같은 지시는 모델이 전체를 갈아엎게 만든다 — 반드시 유지/수정/금지를 나눠서 지시한다.
> DESIGN_REVIEW_PROMPT.md 검수를 먼저 돌렸다면 그 출력의 "재작업 지시문"을 그대로 쓰면 된다.

---

## 재작업 지시 템플릿 (복붙용)

```
재작업 지시. 전체를 다시 만들지 말고 아래 항목만 수정하라.

[유지 — 절대 건드리지 말 것]
- (예: 전체 섹션 순서와 개수)
- (예: 히어로 이미지와 그 비율)
- (예: 색 팔레트, 폰트, 버튼 규격)
- (예: 푸터 사업자 정보, 절차 스텝 섹션)

[수정 1]
- 위치: (예: 히어로 헤드라인)
- 현재: (예: "당신의 소중한 시계를 위한 최고의 선택" — 22자, 추상적)
- 변경: (예: "명품시계, 오늘 바로 매입" — 정체성이 즉시 읽히게, ≤15자)
- 완료 판정: (예: 375px에서 한 줄로 표시되고 서비스가 무엇인지 읽힘)

[수정 2]
- 위치:
- 현재:
- 변경:
- 완료 판정:

(필요한 만큼 반복 — 한 번에 최대 10개까지만. 10개가 넘으면 재작업이 아니라 재설계다.)

[금지]
- 지시하지 않은 섹션·색·폰트·문구 변경 금지.
- 새 섹션/새 색/새 효과 추가로 해결하려 하지 말 것.
- 수정 항목 외의 코드 리팩토링 금지.

[제출 조건]
- 375 / 768 / 1280px에서 잘림·겹침·가로스크롤 없음 확인 후 제출.
- 보고 형식: 수정 항목별로 "완료 판정 기준 충족 여부"를 한 줄씩. 못 고친 항목은 이유 명시.
```

## 자주 쓰는 수정 지시 문구 사전 (추상 지시 → 실행 지시 번역)

| 이렇게 말하고 싶을 때 | 이렇게 지시한다 |
|---|---|
| 촌스러워 | 배경 그라데이션 제거→단색, 색을 6개 이내로 줄이고 CTA색 1개만 유지, 그림자 `0 1px 3px rgba(0,0,0,.06)` 이하로 |
| 글씨가 많아 | 폴드 위 텍스트 4줄 이하, 섹션 본문 3문장 이하로 자르고 나머지는 접기/하위 페이지로 이동 |
| 허전해 | 텍스트를 늘리지 말고 실물 이미지를 키우거나(폭 100%) 핵심 숫자 1개를 크게 배치 |
| 정신없어 | 폰트 크기를 28/20/16/14/12 5단계로 통일, 좌측 시작선 3종 이내로 정렬, 섹션 간격 48~64px 통일 |
| 템플릿 같아 | 아이콘 3열 섹션을 실제 사진/실제 절차/실제 숫자 섹션으로 교체, 이모지·AI 아이콘 제거 |
| 안 눌러볼 것 같아 | CTA를 폴드 위로, 라벨을 "행동+대상"(매입 신청/정비 예약)으로, 페이지 끝에 CTA 재노출+전화 버튼 |
| 못 믿을 것 같아 | 절차 3~5스텝 시각화 추가, 검증 가능한 숫자만 사용, 과장 문구 삭제, 사업자 정보 푸터 노출 |
| 모바일이 이상해 | 375×812 스크롤 0 스크린샷 기준으로 폴드 4요소 재배치, min-width:0/nowrap 점검, 3폭 재확인 |

## 재작업 판단 기준
- 수정 항목 ≤10개 → 이 템플릿으로 재작업.
- 수정 항목 >10개 또는 루브릭 70점 미만 → 부분 수정으로 살리지 말고 "구조 재설계"를 지시한다: 섹션 목록과 폴드 구성을 먼저 텍스트로 제안받고, 승인 후 제작.
- 같은 항목이 2회 재작업에도 안 고쳐지면 → 그 항목의 원하는 결과를 레퍼런스 캡처로 제시한다(말 대신 그림).
