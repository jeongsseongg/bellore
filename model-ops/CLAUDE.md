# CLAUDE.md — Claude Code 프로젝트 공통 지침 (템플릿)
> ⚠️ 이 파일은 **새 프로젝트 루트에 복사해서 쓰는 템플릿**이다.
> 벨로르 레포에는 이미 운영용 루트 CLAUDE.md가 있으므로 이 파일로 덮어쓰지 말 것.
> 새 프로젝트에 넣을 때: 이 파일을 프로젝트 루트에 `CLAUDE.md`로 복사하고, 맨 아래 [프로젝트 개별 정보]를 채운다.

## 작업 원칙 (최우선 — 위반 시 그 작업은 실패로 간주)
1. **정확히 이해 못했으면 추측하지 말고 질문한다.** 요구사항이 100% 확실하지 않으면 질문으로 확인한 뒤 작업한다.
2. **하라는 것만 한다.** 시키지 않은 파일 수정·기능 추가·리팩토링 금지. 하지 말라는 것 절대 금지.
3. **수정은 끝까지.** 폼 필드 하나를 바꾸면 그 필드가 지나가는 모든 지점(payload → DB 저장 → 목록/상세 표시 → CSS → 서버 검증)을 전부 연동한다. 중복 코드는 지적받기 전에 먼저 찾아 단일 소스로 정리한다.
4. **"안 됨"의 원인을 구분해서 보고한다.** "DB 마이그레이션 실행 전엔 동작 안 함", "서버 함수 재배포 필요" 등 선행 조건을 분명히 안내하고, 가능하면 클라이언트 폴백을 넣는다.
5. **부분·추측 결과물로 배포하지 않는다.** 여러 번 확인해서 최대 달성률일 때만 배포한다.

## 작업 전 분석 (코드 수정 전에 반드시)
1. 관련 파일을 **먼저 읽는다.** 읽지 않은 파일의 구조를 추측해서 수정하지 않는다.
2. 프로젝트에 `DESIGN.md` / `DESIGN_DECISION_RULES.md`가 있으면 읽고 그대로 따른다. 디자인 표준에서 벗어나는 판단을 스스로 하지 않는다.
3. 수정 범위를 선언한다: "수정할 파일: A, B / 건드리지 않을 파일: C(이유)". 선언한 범위 밖 파일을 수정해야 하면 이유를 먼저 보고한다.
4. 기존 클래스·토큰·컴포넌트를 재사용한다. 임의 인라인 스타일·일회용 색상·새 유틸 함수 남발 금지.

## 디자인 방향 유지
- 기존 화면의 룩을 바꾸지 않는다. 새 화면은 기존 화면과 같은 규격(버튼 높이, 라운드, 폰트, 간격)을 쓴다.
- 레퍼런스 이미지가 주어지면 색·레이아웃·구성요소를 **그대로** 맞춘다. "비슷하게"가 아니라 픽셀 대응.
- 흑백화/톤 다운/임의 색 변경 금지. 레퍼런스가 컬러면 컬러로 간다.
- 상세 판단 기준은 `DESIGN_DECISION_RULES.md`를 따른다.

## 모바일 / PC 검수 (제출·배포 전 필수)
- **375px / 768px / 1280px** 3개 폭에서 확인한다. 방법 우선순위:
  ① 실제 구동(로컬 서버 + 브라우저) ② 격리 렌더(해당 컴포넌트만 HTML로 띄워 확인) ③ 코드 정독(최후 수단, 보고에 "실구동 미확인" 명시).
- 확인 항목: 잘림·겹침·가로스크롤·폴드 위 CTA 노출·터치 타깃(최소 44px)·고정 헤더/바텀바에 콘텐츠 가림 없음.
- 알려진 함정:
  - flex 자식 잘림 = `min-width:auto` 기본값. 입력칸+버튼 한 줄이면 입력칸 `min-width:0`, 버튼 `white-space:nowrap`.
  - 오버레이 z-index 고정값 겹침 주의. 새 오버레이는 기존 최상위 값을 확인 후 결정.
  - `MutationObserver`로 `document.body` 전체를 감시하며 콜백에서 감시 대상을 수정하는 코드 금지(무한루프/초기화 충돌).
- 대규모·전역 UI 변경은 실제 구동 확인 전 배포 금지.

## 배포·안전 규칙
- 도메인/DNS/CNAME/Pages·호스팅 설정은 명시 지시 없이 절대 건드리지 않는다. (삭제 시 사이트 즉시 다운 사례 있음.)
- 결제·인증·DB·보안 관련 변경은 되돌리기 쉬운 작은 단위로 나누고, 서버 측 검증(금액 재계산 등)을 약화시키지 않는다.
- Service Worker가 있는 프로젝트는 클라이언트 파일(JS/HTML/CSS) 변경 시 캐시 버전을 올린다.
- SQL/마이그레이션은 복붙용 전체 블록으로 제공한다.
- 커밋·PR·코드·주석에 모델 식별자를 넣지 않는다. PR은 명시 요청 시에만 만든다.
