# EXAMPLES.md — 좋은/나쁜 결과물 조건과 페이지 구조 예시

## 1. 좋은 결과물의 조건 10개
1. 375px 스크롤 0 상태에서 "무엇을 하는 곳 + 실물 이미지(또는 핵심 숫자) + CTA 1개"가 다 보인다.
2. 1차 CTA가 페이지에 1종류이고, 라벨이 행동+대상("매입 신청", "정비 예약")이다.
3. 화면의 절반 이상이 실물 사진이고, 텍스트는 그 사진을 설명하는 최소량이다.
4. 폰트 크기 5단계(28/20/16/14/12), 색 6개 이내, 같은 요소끼리 규격이 픽셀 단위로 같다.
5. 모든 섹션이 전환/신뢰/정보/탐색 중 하나의 목적을 갖고, 같은 레이아웃 패턴이 반복되지 않는다.
6. 절차가 3~5단계 스텝으로 시각화되어 있고, 각 스텝 설명이 1줄이다.
7. 숫자(가격, 처리 기간, 실적)가 주변 텍스트보다 크고, 전부 검증 가능한 사실이다.
8. 375/768/1280 어디서도 잘림·겹침·가로스크롤이 없고, PC에서 콘텐츠가 중앙 컬럼에 정돈된다.
9. 페이지 끝에 CTA 재노출 + 전화/카카오 대체 연락 수단이 있다.
10. 경쟁사 이름을 붙여도 어색한, 그 업종·그 브랜드만의 콘텐츠(실제 사진·실제 절차·실제 정책)로 채워져 있다.

## 2. 나쁜 결과물의 조건 10개
1. 첫 화면이 인사말·소개 문단으로 시작하고 CTA가 폴드 아래에 있다.
2. "문의하기/상담하기/신청하기"가 서로 다른 스타일로 한 페이지에 공존한다.
3. 보라→파랑 그라데이션 배경, 유리효과 카드, 떠다니는 그림자 등 AI 기본 출력 스타일이 남아 있다.
4. 아이콘 3열 + 제목 + 설명 패턴이 2번 이상 반복된다(어느 업종에 붙여도 말이 되는 화면).
5. 이모지·AI 생성 티 나는 아이콘·외국인 악수 스톡 사진을 쓴다.
6. "업계 최고가 보장", "100% 만족", "무조건" 같은 과장 문구가 있다.
7. 폰트 크기가 6종류 이상 뒤섞이고, 버튼 높이가 화면마다 다르다.
8. 특정 폭에서 텍스트가 잘리거나 버튼이 어색하게 줄바꿈되거나 가로스크롤이 생긴다.
9. lorem ipsum, "제목을 입력하세요", `href="#"`, 깨진 이미지가 남아 있다.
10. 예쁘지만 이 섹션이 문의·예약·구매에 어떻게 기여하는지 답할 수 없다.

## 3. 페이지 유형별 좋은 구조 예시
> 아래 구조는 "섹션 순서 + 각 섹션의 목적"이다. 섹션을 임의로 늘리지 말고 이 뼈대에서 시작한다.

### 3-1. 랜딩페이지 (단일 목적 — 예: 오토픽스코리아 사고차 매입)
1. **히어로 (전환)** — 실차 사진 풀폭 + 헤드라인 "사고차, 전국 어디든 방문 매입"(≤15자) + CTA "매입 신청" + 보조 1줄 "국내·해외 바이어 견적 비교".
2. **핵심 숫자 (신뢰)** — 큰 숫자 2~3개: 처리 기간(예: "평균 O일 이전 완료"), 출장 범위("전국"), 비교 견적 수. 검증 가능한 것만.
3. **절차 스텝 (신뢰)** — "① 신청 → ② 출장 방문·확인 → ③ 국내/해외 바이어 견적 비교 → ④ 계약·정산 → ⑤ 이전 처리". 각 1줄. 고객 불안 포인트(명의 이전, 대금 시점)를 스텝 안에서 해소.
4. **견적 비교 근거 (정보)** — 왜 견적이 다른지(수출 수요 등)를 3문장 이내 + 실제 매입 사례 카드 2~3개(차종·상태·처리 기간).
5. **자주 묻는 질문 (정보)** — 접기식 4~6개. 이전/압류/사고이력 등 불안 질문 우선.
6. **마감 CTA (전환)** — CTA 재노출 + 전화 원터치 + 카카오톡. 푸터에 사업자 정보.

### 3-2. 커머스 메인 (벨로르 — 명품시계)
1. **헤더** — 로고 + 검색 + (알림/장바구니). 얇게.
2. **히어로/배너 (전환·탐색)** — 시계 실물 대형 사진 배너 1~3장 스와이프. 배너 문구 ≤10자.
3. **카테고리 바로가기 (탐색)** — 원형 컬러 아이콘 한 줄(브랜드/매입/위탁/감정). 레퍼런스(바이버/구구스) 규격 그대로.
4. **상품 그리드 (전환)** — 2열 카드: 사진(카드의 60%+) → 브랜드·모델(1줄) → **가격(가장 크게)** → 상태 뱃지. 큐레이션 제목("이번 주 입고") 붙여 3~4행.
5. **내 시계 팔기 유도 (전환)** — 밴드 1개: "내 시계 얼마에 팔릴까?" + CTA "매입 시세 조회". 매입은 커머스의 제2 전환이므로 메인 중단에 1회 노출.
6. **신뢰 밴드 (신뢰)** — 정품 보증·가품 시 전액환불·안전거래를 아이콘 없이 짧은 문장 3개로.
7. **푸터** — 법적 문서 5링크 + 사업자 정보.

### 3-3. 정비센터 홈페이지 (프로모터스 — 수입차 정비)
1. **히어로 (전환)** — 실제 작업장/리프트 위 수입차 사진 풀폭(어두운 오버레이) + "수입차 정비, 예약하고 오세요"(≤15자) + CTA "정비 예약". 전화 버튼을 CTA 옆 2차로.
2. **취급 브랜드/정비 항목 (정보)** — 로고 또는 텍스트 한 줄 나열 + 대표 정비 항목 카드 4~6개(엔진/미션/브레이크/판금 등, 각 1줄).
3. **정비 사례 (신뢰)** — 실제 작업 전/후 사진 카드 3~6개: 차종 + 작업 내용 1줄 + 기간. 이 섹션이 이 페이지의 신뢰 엔진이므로 사진 품질 최우선.
4. **이용 절차 (신뢰)** — "예약 → 입고·진단 → 견적 안내 → 정비 → 출고". 견적 먼저 안내 후 작업함을 명시(과잉정비 불안 해소).
5. **오시는 길 (정보)** — 지도 + 주소 + 주차 안내 + 영업시간. 전화 원터치.
6. **공지/소식 (정보)** — 최근 3건 리스트. 없으면 섹션 자체를 뺀다(빈 공지가 더 나쁨).
7. **마감 CTA (전환)** — "정비 예약" 재노출 + 전화. 푸터 사업자 정보.

### 3-4. 명품시계 매입 페이지 (시계상담소/뉴욕워치)
1. **히어로 (전환)** — 시계 클로즈업 실사 + "내 시계, 지금 시세 확인"(≤15자) + CTA "매입 상담". 보조 1줄 "사진만 보내도 예상 시세 안내".
2. **시세 안내 (정보·신뢰)** — 핵심 메시지: **검색 시세와 실제 매입가는 다르다.** 3문장 이내로 이유(리테일가/중고 호가/실거래가 차이) + "높게 부르고 나중에 깎는 곳 주의" 를 사실 톤으로. 여기서 정직함이 차별점이 된다.
3. **감가 요인 (정보)** — 표 또는 카드: 구성품(박스/개런티카드/보증서), 상태(폴리싱·기스), 연식/스탬핑, 인기 모델 여부 → 각각 시세에 미치는 영향 1줄. 고객이 자기 시계를 스스로 가늠하게 만든다.
4. **매입 절차 (신뢰)** — "상담(사진) → 예상 시세 → 방문/택배 감정 → 확정 견적 → 즉시 정산". 확정 견적 후 감액 없음(정책일 때만 명시).
5. **매입 사례 (신뢰)** — 실물 사진 + 모델명 + 구성품 상태 + 매입가 표기 가능 범위 내 사례 3~6개.
6. **마감 CTA (전환)** — "매입 상담" 재노출 + 카카오톡/전화. 푸터 사업자 정보.

## 4. 구조 사용 규칙
- 위 뼈대에서 섹션을 **빼는 것**은 허용(콘텐츠가 없으면 뺀다), **추가는 목적을 적을 수 있을 때만**.
- 순서 변경은 히어로(1번)와 마감 CTA(마지막)를 제외하고만 허용.
- 모든 유형 공통: 히어로에 실물 사진, 절차 스텝 1개 섹션, 마감 CTA + 대체 연락 수단, 푸터 사업자 정보 — 이 4개는 생략 불가.
